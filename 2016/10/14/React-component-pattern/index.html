React中的组件分类

原文链接

React 中一切都是组件化的, 把组件分为2种类型有利于开发和重用. 可以把这两种类型称为容器组件(Container component) 和 呈现组件(Presentational component).

它们有什么区别呢?

呈现组件的特点:

  • 关注UI的展现
  • 可以在内部包含呈现和容器组件, 通常有一些DOM标签和自己的样式
  • 仅从props接收数据和回调
  • 通常没有自己的state
  • 通常可以写成函数组件, 除非需要 state 或生命周期函数
  • 和应用中其他部分没有关联, 比如 Redux
  • 不关心数据是怎样传递和修改的
  • 可以有this.props.children
  • 通常是手写的
  • 例如: Page, Sidebar, Story, UserInfo, List

容器组件的特点:

  • 关注业务逻辑
  • 可以在内部包含呈现和容器组件, 但通常自身没有DOM标签(除了外层的包含div), 也没有样式
  • 给呈现组件和其他容器组件提供数据和方法
  • 通常有自己的state
  • 可以和 Redux 联系, 通常由 Redux 的connect()函数生成, 从而可以接收来自 Redux 的State的数据以及 dispatch Redux actions
  • 可以调用 Redux 的 actions ,然后作为回调传递给呈现组件
  • 例如: UserPage, FollowersSidebar, StoryContainer, FollowedUserList

在开发时可以把它们放在不同的目录中加以区别.

组件分类的好处

  • 把关注的内容分离. 这样可以更好地理解你的应用和 UI
  • 组件重用. 可以用同样的呈现组件和不同的数据源搭配, 变成各自独立的容器组件, 之后便可以重用
  • 呈现组件是应用中的’调色板’. 你可以把它们放在一个单独页面中让设计师来修改, 而不会接触到应用的逻辑. 这样方便在那个页面中回溯不同的版本.
  • 这样可以强迫你把’布局组件’例如Sidebar, Page, ContextMenu等提取出来, 并使用this.props.children, 而不是在几个容器组件中复制同样的内容和布局

记住一点, 组件不一定要产生DOM. 它们是 UI 之间的一种组合方式.

什么时候需要容器组件呢?

建议从呈现组件开始写你的应用. 最终你会发现你把很多的 props 传递给了中间组件. 当你注意到一些组件并没有使用它们接收到的 props 而仅仅是把它们传递给下一层组件, 而且任何时候当它们的子组件需要更多数据时你都要修改这些中间组件, 这时候你就应该使用一些容器组件了. 这样子你就可以把数据和方法传递给叶子组件而不需要关心树中间那些不相关的组件.

呈现组件和容器组件有各自产生的方式. 呈现组件趋向于使用无状态的纯函数, 容器组件趋向于有状态的纯 class. 但这只是一种观点而不是准则, 在特定情况下也可能会出现相反的情况 .
不要把呈现组件和容器组件的分离当成教条. 有时候这并不重要, 或者你很难去区分. 如果你不确定一个组件应该是哪一种时, 可能还没到该决定的时候.

例题

这里是最初未分离的代码 

// CommentList.js

class CommentList extends React.Component {
  constructor(props) {
    super(props);
    this.state = { comments: [] }
  }

  componentDidMount() {
    $.ajax({
      url: "/my-comments.json",
      dataType: 'json',
      success: function(comments) {
        this.setState({comments: comments});
      }.bind(this)
    });
  }

  renderComment({body, author}) {
    return <li>{body}—{author}</li>;
  }

  render() {
    return <ul> {this.state.comments.map(renderComment)} </ul>;
  }

}

下面分离成2个组件, 第一个是呈现组件, 第二个是容器组件 

// CommentList.js

class CommentList extends React.Component {
  constructor(props) {
    super(props);
  }

  renderComment({body, author}) {
    return <li>{body}—{author}</li>;
  }

  render() { 
    return <ul> {this.props.comments.map(renderComment)} </ul>;
  }

}

// CommentListContainer.js

class CommentListContainer extends React.Component {
  constructor() {
    super();
    this.state = { comments: [] }
  }

  componentDidMount() {
    $.ajax({
      url: "/my-comments.json",
      dataType: 'json',
      success: function(comments) {
        this.setState({comments: comments});
      }.bind(this)
    });
  }

  render() {
    return <CommentList comments={this.state.comments} />;
  }
}
