Vue 源码解读-响应式系统

Vue 的核心之一就是它的响应式系统, model 只是普通的 JavaScript 对象, 当修改它时, view 会进行更新. 当把一个对象作为 data 属性传入 Vue 的实例中时, Vue 会遍历它上面的属性, 并用 Object.defineProperty 把它们转为 getter/setter, 以进行依赖追踪和通知更改. Vue 实现响应式的原理代码位于 src/core/observer, 目录结构如下, 以下基于 Vue 2.1

Vue 内部实现响应式时使用了 发布-订阅 的设计模式, 主要内容为 3 大块: Observer, Watcher, Dep. Observer 使 data 上的数据成为发布者, 而指令(包括模板指令, computed, $watch 内的属性)对应的 watcher 则为订阅者, Dep 的作用主要在于维护订阅者数组, 当数据变化时通知订阅者进行更新.

Observer

Observer 给 data 上的数据设置 getter/setter. 当设置 getter 时, 进行’依赖收集(订阅)’过程, 当 setter 触发时, 进行’通知更新(发布)’的过程.

需要注意的是, 并不是每次求值调用 getter 方法都会收集依赖, 而是当 Dep.target 存在时才会去收集, Dep.target 作为一个全局变量, 它里面存放的是当前的 watcher. 如果 Dep.target 为空, 说明没有指令订阅该数据, 也就不需要进行依赖收集.

用到 new Observer() 的地方只有一个: observe 函数中

Dep

dep 是一个 observable, 指令对应的 watcher 会订阅它, 即 dep 把该 watcher 加入自己的订阅者数组. dep 就是依赖, 依赖的含义是’表达式或函数的值对 data 上数据的依赖’. 依赖收集就是 data 上某个数据对应的 dep 把一个 watcher 加入订阅者数组里面.
在上面的 observe 函数中以及 defineReactive 中设置 getter/setter 前, 会先给属性创建一个 dep 实例以用于依赖收集.

Watcher

watcher 解析表达式的值, 收集依赖, 当表达式的值变化时触发回调, $watch api 和指令都用的这种方式.
前面提到 Observer 使 data 变成发布者, 指令对应的 watcher 则是订阅者, 订阅了 data 上的数据, 当数据变化时, watcher 会通知指令更新自己所在的 DOM 或更新自身的值.

对 computed 属性 c 的求值过程如下:

  1. 新建 Vue 实例时, computed 里的值 c 经历过程如下: new Vue() -> _init() -> initState() -> initComputed() -> makeComputed()
  2. 在 makeComputed 中会为 c 新建一个 watcher, watcher 中 getter 为 cvalue() 函数, 当第一次对 c 求值时, dirty 属性为 true, 会调用 watcher 的 evalute() 进行求值, evalute 中会调用 watcher 的 get(), get() 内通过 pushTarget 先把自身(c 对应的 watcher)设为 Dep.target, 再调用 getter 即 cvalue()
  3. 对 a, b 求值, 触发 a, b 的 getter, getter 内通过 dep.depend() 把 c 加入它们的 subs 数组, 即 c 订阅了 a 和 b, 然后 get() 内 通过 popTarget 恢复原 Dep.target 值(null) , 返回值, watcher.value 的值变为 a + b, dirty 属性变为 false

如果之后 a, b 的值没有变化, 再对 c 求值时, 由于 watcher.dirty 为 false, 会直接返回 c 的值.
如果 a 或者 b 的值变化了, 会通知 c 进行更新, 过程: a 对应的 dep 中 notify() -> 调用 c 的 update() -> dirty 设为 true,
调用 run() -> 重新调用 get() -> 重复上面 get() 的步骤
