redux源码解读(四)-compose

概述

compose这个函数在函数式编程中很常见, 它把一个函数函数的结果传进另一个函数中, 例如compose(f, g, h)(x) 返回 f(g(h(x)))

下面先看一个简化版本的compose以便理解 

function compose(f, g) {
  return function() {
    return f.call(this, g.apply(this, arguments));
  };
  /* 如果只接收一个参数, 也可以这样写
  return function(x) {
    return f(g(x));
  };
  */
}  

const square = x => x * x;
const sum = (x, y) => x + y;
const res = compose(square, sum);
res(2, 3); // 25

redux源码解读(三)-bindActionCreators

概述

  • 函数参数, bindActionCreators接收2个参数, actionCreators和dispatch,
    actionCreators可以是一个函数, 也可以是包含几个函数的对象

  • 返回值, 返回值对应actionCreators这个参数, 当它是函数时, 返回值是一个函数, 这个返回的函数接收一个参数传入actionCreator得到 action, 再调用dispatch; 当它是包含几个函数的对象时, 返回值为包含这几个函数调用结果的对象

bindActionCreators() 是函数式编程中的组合函数compose, 在下一篇的compose中会提到, 组合函数形如
compose(f, g)(...args) --> g(f(...args)), 在这里f和g分别为参数actionCreator和dispatch

React中的组件分类

原文链接

React 中一切都是组件化的, 把组件分为2种类型有利于开发和重用. 可以把这两种类型称为容器组件(Container component) 和 呈现组件(Presentational component).

它们有什么区别呢?

呈现组件的特点:

  • 关注UI的展现
  • 可以在内部包含呈现和容器组件, 通常有一些DOM标签和自己的样式
  • 仅从props接收数据和回调
  • 通常没有自己的state
  • 通常可以写成函数组件, 除非需要 state 或生命周期函数
  • 和应用中其他部分没有关联, 比如 Redux
  • 不关心数据是怎样传递和修改的
  • 可以有this.props.children
  • 通常是手写的
  • 例如: Page, Sidebar, Story, UserInfo, List

JavaScript Promise

A promise is a placeholder for the result of an asynchronous operation. Instead of subscribing to an event or passing a callback to a function, the function can return a promise

Syntax

new Promise( /* executor */ function(resolve, reject) { ... } );

Parameters: executor

A function that is passed the arguments resolve and reject. The executor function is executed immediately by the Promise implementation, passing resolve and reject functions (the executor is called before the Promise constructor even returns the created object). The resolve and reject functions, when called, resolve or reject the promise respectively. The executor normally initiates some asynchronous work and then, once that completes, calls either the resolve or reject function to resolve the promise or else reject it if an error occurred

Calling resolve() triggers an asynchronous operation. Functions passed to then() and catch() are executed asynchronously, because these are also added to the job queue

Each call to then() or catch() actually creates and returns another promise. This second promise is resolved only when the first has been fulfilled or rejected

JavaScript中数组迭代方法的简单实现

JavaScript 中数组迭代方法的简化实现

map

Array.prototype.map = function(fn /*, thisArg*/) {
  var res = [];
  var thisArg;
  if (this == null) {
    throw new TypeError('Array.prototype.map2 called on null or undefined');
  }
  if (typeof fn !== 'function') {
    throw new TypeError(fn + ' is not a function');
  }
  if (arguments.length > 1) {
    thisArg = arguments[1];
  }

  for (let i = 0; i < this.length; i++) {
    if (i in this) {
      res.push(fn.call(thisArg, this[i], i, this));
    }
  }
  return res;
};
