Vue 源码解读-响应式系统

Vue 的核心之一就是它的响应式系统, model 只是普通的 JavaScript 对象, 当你修改它时, view 会进行更新. 当把一个对象作为 data 属性传入 Vue 的实例中时, Vue 会遍历它上面的属性, 并用 Object.defineProperty 把它们转为 getter/setter, 以进行依赖追踪和通知更改. Vue 实现响应式的原理代码位于 src/core/observer, 目录结构如下

Observer

Observer 把目标对象(比如 data) 的属性 keys 转为 getter/setter, 进行依赖收集(setter)和发布更新(setter). 当设置 getter 时, 进行’依赖收集(订阅)’过程, 当 setter 触发时, 进行’通知更新(发布)’的过程. 但并不是每次 getter 方法都会收集依赖

Dep

dep 是一个 observable, 指令(watcher)会订阅它. dep 就是’依赖’, 依赖的含义是’表达式或函数的值对 data 的依赖’.

watcher

看一下源码中的对 watcher 的描述:
A watcher parses an expression, collects dependencies, and fires callback when the expression value changes. This is used for both the $watch() api and directives.

前面提到 Observer 使 data 变成’发布者’, 而’指令’对应的 watcher 则是订阅者, 订阅了 data, 当 data 变化时, watcher 会通知’指令’更新自己所在的 DOM.
